# 单向数据流实现计划

## 方案选择：React Context + useReducer

**理由**：项目规模小（~5 个源文件），Redux Toolkit / Zustand 会引入不必要的依赖。`useReducer` 是 React 内置的 dispatch-based 状态管理方案，零额外依赖，完美匹配单向数据流模式。

## 核心设计

### 状态模型

```ts
// 当前：4 个独立 useState，selectedRecords 需手动从 localStorage 同步
// 目标：单一 reducer state，records 持有全量数据，selectedRecords 派生

interface AppState {
  currentDate: string;           // 当前选中日期
  records: RideRecord[];         // 全量记录（不再从 localStorage 反复读取）
  showForm: boolean;             // 表单弹窗可见性
  editingRecord: RideRecord | null; // 正在编辑的记录
}

// 派生数据（selector）：
// selectedRecords = records.filter(r => r.date === currentDate)
// totals = { duration, distance, expense } — 从 selectedRecords 计算
```

### 数据流

```
用户操作 → dispatch(action) → reducer 纯函数计算新 state
  → Provider useEffect 监听 records 变化 → 自动写入 localStorage
  → React 重渲染 → selector 派生 filtered/total 数据
```

消除了当前 "调用 storage → 手动 re-read → setState" 的两步同步模式。

## 文件结构

```
src/
  store/
    types.ts        # AppState, AppAction 类型定义
    reducer.ts      # 纯 reducer 函数
    selectors.ts    # 派生数据计算（selectedRecords, totals）
    context.tsx     # Provider 组件 + useAppState/useAppDispatch hooks
```

## 实施步骤

### Step 1 — 创建 `src/store/types.ts`

定义 `AppState` 接口和 `AppAction` 联合类型：

| Action | Payload | 说明 |
|--------|---------|------|
| `SET_DATE` | `string` | 切换日期 |
| `ADD_RECORD` | `RideRecord` | 新增记录 |
| `UPDATE_RECORD` | `{ id, data }` | 更新记录 |
| `DELETE_RECORD` | `string` (id) | 删除记录 |
| `OPEN_FORM` | `RideRecord?` | 打开表单（有值=编辑，无值=新增） |
| `CLOSE_FORM` | — | 关闭表单 |

### Step 2 — 创建 `src/store/reducer.ts`

纯函数 `(state, action) → newState`。每个 case 通过 spread 返回新对象。无副作用，不调用 localStorage。

### Step 3 — 创建 `src/store/selectors.ts`

- `selectRecordsByDate(records, date)` — 替代 `storage.getRecordsByDate`
- `selectTotals(records, date)` — 替代 App.tsx 中的 3 个 `.reduce()`

### Step 4 — 创建 `src/store/context.tsx`

- Provider 组件：初始化时从 `storage.getRecords()` 加载全量数据
- `useReducer(appReducer, initialState)`
- 拆分 Context：`AppStateContext` + `DispatchContext`（避免 dispatch-only 消费者不必要的重渲染）
- `useEffect` 监听 `state.records` 变化 → 调用 `saveRecords()` 写入 localStorage（write-through 持久化）
- 导出 `useAppState()` 和 `useAppDispatch()` 便捷 hooks

### Step 5 — 重构 `src/App.tsx`

- 移除 4 个 `useState` 和 `useEffect`
- 移除 `storage` 直接导入
- 用 `useAppState()` 获取 state，用 `useAppDispatch()` 派发 action
- 用 selector 派生 `selectedRecords` 和 totals
- 事件处理函数改为 dispatch 调用

### Step 6 — 更新 `src/main.tsx`

- 用 `<AppProvider>` 包裹 `<App />`

### Step 7 — 运行 `npm run build` 验证

确保 `tsc` 无类型错误，`vite build` 成功。
